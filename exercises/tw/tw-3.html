Documentation Layout with Tailwind (CDN)

This page demonstrates a simple, production-ready documentation layout built with Tailwind via the CDN. On large screens (lg and up), you get a left sidebar with section links; on smaller screens, everything stacks into a single column for a clean reading experience.

Installation

Using the CDN is the fastest way to experiment. Add the script tag below to your HTML and include the Typography plugin to get pleasant default styles for prose content.

<script src="https://cdn.tailwindcss.com?plugins=typography"></script>

Layout

The layout uses a responsive grid: a fixed-width sidebar and a flexible content column. The main article is constrained with max-w-prose for optimal line length and comfortable reading.

  • lg:grid-cols-[16rem_minmax(0,1fr)] creates the two-column layout on large screens.
  • The sidebar is hidden by default and shown with lg:block.
  • Headings get generous top margins using utility classes like mt-12.

Components

The “Edit on GitHub” button is a utility-based component: an anchor tag styled entirely with Tailwind classes. Copy the markup from the header and replace the href with your repository’s edit URL.

<a href="https://github.com/your/repo/edit/main/path/to/page.md" class="inline-flex items-center gap-2 rounded-lg border border-slate-300 bg-white px-3 py-2 text-sm font-medium text-slate-700 shadow-sm transition hover:-translate-y-0.5 hover:bg-slate-50 hover:shadow dark:border-slate-700 dark:bg-slate-900 dark:text-slate-200 dark:hover:bg-slate-800">Edit on GitHub</a>

Accessibility

There’s a “Skip to content” link for keyboard users, sensible aria-labels, and focusable targets with visible hover states. The sidebar navigation is a semantic <nav>, and the mobile table of contents uses a native <details> disclosure.
